# Roadmap

**Stack:** Node.js 20+ · Express 4 · EJS 3 · TypeScript 5 · Tailwind CSS 4 · PostCSS

| No | Feature / Module | Description | Why Needed | EJS / Express Implementation Notes | Priority | Coverage |
|----|-----------------|-------------|------------|-------------------------------------|----------|----------|
| 1 | **Partial Variant System** | Each EJS partial accepts a `variant`, `size`, and `intent` local and maps it to a class string via a `const variantClasses` object at the top of the partial | Prevents scattered one-off className strings duplicated across themes; enforces token-aligned visual consistency | Define `modules/ui/Button.ejs` to expect `locals.variant` (default `'primary'`); top of file: `<% const v = variant \|\| 'primary'; const cls = {primary:'bg-primary text-primary-fg', secondary:'...'} %>` then `class="<%= cls[v] %>"` | Must-have | ✅ Complete — tüm `modules/ui/` partial'larında `_v`/`_sz`/`vc`/`sc` lookup tabloları standartlaştırıldı; `bg-surface-active`, `text-text-tertiary`, `ring-primary/20`, `hover:border-text-tertiary` gibi sahte token kullanımları token-aligned karşılıklarıyla değiştirildi |
| 2 | **Polymorphic Element Override** | EJS partials accept an `element` local (`'a'`, `'button'`, `'div'`) so the same partial renders as different HTML elements without wrapper markup | Reuse `Button.ejs` as an anchor when `href` is provided, without a second template | In `Button.ejs`: `<% const el = element \|\| 'button'; %>` then `<<%= el %> ...attrs...>` — pass `element: 'a'`, `href: '/path'` from the include call | Must-have | ✅ Complete — `Button.ejs` `element` local'ı ile `<a>`, `<button>` vb. render ediyor; `href`, `target`, `rel`, `aria-disabled` otomatik yönetiliyor |
